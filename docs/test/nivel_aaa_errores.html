Documentación Técnica del Sistema

La implementación del API REST utiliza JWT para la autenticación de usuarios. El protocolo HTTP define las cabeceras de autorización que el cliente debe incluir en cada solicitud. La especificación WCAG establece que las aplicaciones web deben ser accesibles para todos los usuarios, incluyendo aquellos con discapacidades visuales, auditivas o motoras que requieren el uso de tecnologías asistivas como lectores de pantalla o dispositivos de entrada alternativos al teclado y ratón convencionales.

El sistema solo acepta solicitudes con cabeceras CORS correctamente configuradas, mas puede operar en modo degradado cuando el servidor externo no responde. Aun cuando las credenciales sean válidas, el token JWT debe renovarse periódicamente como medida de seguridad. La URL del endpoint debe incluir el identificador único del recurso donde se almacena la información.
